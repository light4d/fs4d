# fs4d 设计需求

fs4d，要实现一种简单、可维护性高，又支持分布式规模集群存储的文件服务。

## 对标系统

hdfs

## 内部设计
借助硬盘文件系统自身的可扩展性，利用文件，作为一种索引，来组织时空对象。

## 访问划分

rest访问接口

+ GET 
获取文件，或文件夹列表
+ POST
上传文件
+ HISTORY
查看历史版本
+ DELETE
删除
